# 复盘端

**定位**：独立 Skill；对单次发布（或单次写作+发布）做**全面**留痕与沉淀，便于回看「这条为什么发、用的素材是什么、失败在哪一步」并拉取互动数据、产出可复用经验。

**说明**：与写作流程的「发布与反馈」无强耦合；若会话目录来自写作流程，则自动利用其中间件做全面追溯；否则做最小化留痕。

---

## 输入

- **会话目录**：用户或调用方提供的路径。目录内预期存在：
  - **必读**：`09-发布记录.md`（来自发布端）。
  - **可选写作中间件**：`00-需求解析与约束.md`、`01-素材清单与清洗.md`、`02-素材索引与受众.md`、`03-主张与支撑点.md`、`04-大纲与段落计划.md`、`05-目的覆盖后段落计划.md`、`06-母稿与引用映射.md`、`07-平台交付稿.md`、`08-发布与反馈检查.md`、`正文.md`。
- 若为「仅发布记录」的轻量复盘（无写作中间件），则仅基于 `09-发布记录.md` 与 `正文.md` 等已有文件做最小留痕。

---

## 复盘内容（全面）

产出写入该会话目录下 **`10-复盘留痕.md`**，且须覆盖以下**全部小节**（无数据时填「无」或「不适用」）：

1. **会话摘要**：会话目录名、任务主题、写作目的、目标平台、创建/发布时间范围。
2. **发布决策追溯**：为什么要发这条（任务卡目的、目标受众、成功指标）；是否来自写作流程、从哪一步产出定稿。
3. **素材与主张追溯**：使用的素材来源（引用 `02`、`01` 的要点或路径）；主张与支撑点（引用 `03`）；大纲或段落计划要点（引用 `04`、`05`）；母稿与引用映射摘要（引用 `06`）。便于回答「用的素材是什么」。
4. **发布结果与失败步骤**：发布是否成功；若失败，失败发生在哪一步（网络/限流/鉴权/重复等）、错误信息、重试次数；若成功，tweet_id、推文链接、正文摘要。
5. **互动数据**：若已配置 Twitter Engagement API 或等效工具且存在 tweet_id，拉取该条推的 impressions、likes、retweets、replies、engagement rate 等并写入；若未配置或未发成功，标注「待拉取」或「未配置」。
6. **交付稿与最终正文**：本次发布的最终正文摘要或路径引用；多平台时标注推特部分。
7. **风险与合规**：本次任务卡中的风险前置清单或合规要点（若有）；发布前检查结论（若有 `08`）。
8. **经验沉淀与资产库建议**：基于本次表现（互动数据、主张与结构）给出可写回资产库的建议（写作目的、平台、受众阻力类型；触发条件、可替换槽位、示例句式或风险提示）；或引用 `08-发布与反馈检查.md` 中的资产库更新说明；无则写「暂无建议」。
9. **后续可选项**：建议的后续动作（如隔日再拉互动、A/B 对比、同主题续写等）；可选。

---

## 拉取互动数据

- 若存在 tweet_id 且环境已配置 Engagement API 或脚本，则执行拉取并填入「互动数据」小节；未配置则不报错，仅在该小节标注「待拉取」或「未配置」。

---

## 产出

- **`10-复盘留痕.md`**：写入指定会话目录，包含上述 1～9 节完整内容。

---

## 质量门禁

- [ ] `10-复盘留痕.md` 已生成并写入会话目录。
- [ ] 上述 1～8 节均有填写（可为「无」或「不适用」）。
- [ ] 「互动数据」节若未拉取，已明确标注「待拉取」或「未配置」。

**全部打勾后**，本 Skill 为单次复盘终点；可由调用方决定是否合并到统一复盘索引或知识库。

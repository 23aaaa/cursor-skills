# 发布端

**定位**：独立 Skill；不依赖写作主调度。由用户或 OpenClaw 在「有交付稿/正文并要发到 X」时调用。

**说明**：需配置 MCP 发推工具（如 x-twitter-mcp、x-twitter-posting-mcp）或提供项目内脚本 `scripts/post_to_twitter.py`，否则仅能产出「待发布」记录。发布端会**协助用户安装所需 MCP** 并**获取所需材料**，再执行发布。

---

## 前置：MCP 安装与材料获取

- **安装所需 MCP**：在发布前协助用户确认或安装发推用 MCP（如 x-twitter-mcp、x-twitter-posting-mcp）。可参考项目根目录 `mcp.json`、`README-MCP-X.md` 的配置与说明；若用户环境未配置，给出安装/启用步骤（如 Cursor MCP 配置方式、所需环境变量）。
- **获取所需材料**：确认用户具备：(1) 发推所需凭证或 API 配置（如 MCP 要求的环境变量、密钥）；(2) 会话目录路径或交付包位置；(3) 待发正文文件（见下方「输入」）。缺项时提示用户补充或引导到可获取处。

---

## 输入

- **会话目录**：用户或调用方提供的路径（如 `写作创作/YYYYMMDD-HHMM-主题简写/`）。目录内预期存在：
  - `正文.md` 或 `正文-推特.md` 或 `07-平台交付稿.md`（推特部分）作为待发内容；
  - 可选 `00-需求解析与约束.md`（任务卡）用于记录主题。
- 若目录内无 `07-平台交付稿.md`，则以 **`正文.md`**（或 **`正文-推特.md`**）作为待发内容。

---

## 发布执行

- **调用方式**：优先使用 MCP 工具（x-twitter-mcp / x-twitter-posting-mcp）的 `post_tweet`、`post_thread`；若无则执行项目内脚本 `scripts/post_to_twitter.py`。
- **单条 vs 线程**：若交付包为多条线程，按条调用 `post_thread` 或依次 `post_tweet`；否则单条 `post_tweet`。

---

## 重试与限流（执行约定）

- **失败重试**：对 429/5xx 进行有限次数重试（建议 2～3 次），等待时间尊重响应头 `x-rate-limit-reset` 或采用退避（如 15s、30s）。
- **限流**：发前若可获取「今日已发条数」，接近上限（如 1～2 条/天）则等待或跳过并记录原因。
- **重复发布**：同一会话目录或同一正文 hash 在短时间窗口（如 24h）内不重复发；若检测到重复则跳过并记录「已跳过-重复」。

---

## 产出

- **发布记录**（结构化）：会话目录、主题/任务卡摘要、发布时间或跳过原因、tweet_id（成功时）、正文摘要或 hash、失败原因与步骤（失败时）、重试次数。
- **落盘**：在该**会话目录**下写入 **`09-发布记录.md`**（Markdown 结构化，便于人与复盘端读取）。

---

## 质量门禁

- [ ] 发布动作已执行（成功或明确失败/跳过）。
- [ ] `09-发布记录.md` 已写入指定会话目录。

**全部打勾后**，可由调用方决定是否继续调用 **.cursor/复盘端** 做全面复盘。
